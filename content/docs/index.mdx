## Purpose

This document defines **why the system exists**, **what it is**, and **how every layer works** — with sufficient precision for a competent engineer to build, maintain, and extend it without additional clarification.

At its core, Sentinel AI is an **offline-capable incident-ingestion, processing, and alerting backend**. It can run either as a full central control-plane or as a lighter edge profile that keeps local SQLite state and replays accepted events upstream when connectivity returns. The live dashboard, WebSocket feed, and dashboard API are operator-facing surfaces layered on top of the central pipeline.
